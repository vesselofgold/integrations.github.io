Integration Delivery Service Team Principles

These principles are for project managers, solution architects, application developers, anyone developing systems to integrate with applications within the Justice and Public Safety infrastructure.

  • All integration services are developed and delivered using agile methodologies
  • Be future friendly by determining re-usability (can this be developed in a way that can be reused by another application)
  • When integration components are developed, modern technology stacks will be used
  • Put clients first by ensuring clear integration requirements are documented and the solution fits those requirements
  • Design for flexibility and speed
  • Design with a digital ecosystem mindset
  • Ongoing engagement with vendor teams/organizations on new tech
  • services deployments should be lightweight and easily manageable
  • open standards and methodologies should be adopted in preference to proprietary solutions for integration
  • mediation between systems (consumers and provider endpoints)
  • service providers and consumers will use standard web service Application Programming Interfaces (APIs) to expose and obtain information from other services and follow the full API management lifecycle
  • when dealing with distributed systems integration should design interactions to be asynchronous to manage problems such as availability and latency
